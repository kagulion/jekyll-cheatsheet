## {{ page.title }}

Jekyll использует специфичную структуру папок для генерации сайта.

- `_config.yml` — Глобальные настройки сайта, такие как название, описание и настройки плагинов.
- `_drafts/` — Черновики постов, которые еще не готовы к публикации (файлы без даты).
- `_includes/` — Фрагменты кода (partials), которые можно вставлять в другие шаблоны с помощью `{% raw %}{% include file.html %}{% endraw %}`.
- `_layouts/` — Шаблоны страниц. Они оборачивают контент, написанный в постах и страницах.
- `_posts/` — Контент блога. Файлы должны иметь формат `YYYY-MM-DD-title.md`.
- `_data/` — Файлы данных (YAML, JSON, CSV). Данные из них доступны через `site.data`.
- `_sass/` — Файлы SASS/SCSS. Их можно импортировать в главном файле стилей.
- `_site/` — Сгенерированный сайт. Эту папку обычно не редактируют и добавляют в `.gitignore`.
- `assets/` — Папка для статических файлов (CSS, JS, изображения).

Подробнее: [Directory Structure](https://jekyllrb.com/docs/structure/)
